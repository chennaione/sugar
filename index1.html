Introducing Sugar Android database made easy.

What does it do?

It is intended to simplify the interaction with SQLite database in android.

  • It eliminates writing SQL queries to interact with SQLite db.
  • It takes care of creating your database.
  • It manages object relationships too.
  • It provides you with clear and simple APIs for db operations

Download

Why it was made?

We felt the need for it while building android apps that had following characteristics..

  • Fairly large and complex data model.
  • Primary work was to save/retrieve and iterate over the business objects.
  • Lot of boiler plate code for the db operations.
  • We wanted to write our business logic in less technical and more business fashion.

What does it provide?

There are two parts to it:

  • database creation
  • simple APIs to manipulate your business objects



Creating tables

Currently:

sampleDB.execSQL("CREATE TABLE IF NOT EXISTS BOOK" + " (title VARCHAR, edition VARCHAR);");

With Sugar: 

public class Book extends SugarRecord{
	String title;
	String edition;

	public Book(Context ctx, String title, String edition){
		super(ctx);
		this.title = title;
		this.edition = edition;
	}
}

Just create the classes extending SugarRecord and it creates a table with class name with its properties as columns.




Save and update data

Currently: 

sampleDB.execSQL("INSERT INTO BOOK" +" Values ('Title here','2nd edition');");

or 

ContentValues values = new ContentValues();
values.put("title", "Title here");
values.put("edition", "2nd edition");
sampleDB.insert("BOOK", null, values);

With Sugar: 

new Book(ctx, "Title here", "2nd edition").save();
